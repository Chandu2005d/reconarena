Passive Reconnaissance

Passive reconnaissance involves collecting information about a target without directly interacting with the target system or network. It uses publicly available sources to build intelligence silently, reducing the risk of detection. This phase is crucial in mapping digital footprints, understanding infrastructure, and planning potential attack vectors.

OSINT Framework

A collection of tools and resources for open-source intelligence gathering. It provides access to services for social media analysis, domain research, and people search without active probing.

Access OSINT Framework

WHOIS Lookup

WHOIS databases reveal domain registration details such as the registrant's name, email, and organization. Useful for identifying administrative contacts or related domains.

Access WHOIS Lookup

DNS Enumeration

Passive DNS methods collect information from public sources or historical records to map subdomains and DNS infrastructure without querying the target directly.

Access DNS Dumpster

Certificate Transparency Logs

These public logs list SSL/TLS certificates issued by Certificate Authorities. Analysts can find subdomains and infrastructure linked to a domain using tools like crt.sh.

Access CRT.SH

Reverse DNS Lookup

Determines the domain name associated with an IP address by checking PTR records. Useful for mapping hosting infrastructure.

Access Reverse DNS Lookup

Email Harvesting

The process of finding exposed or published email addresses linked to an organization. Tools: Hunter.io, theHarvester.

Hunter.io

Hunter.io is a cloud-based OSINT platform that identifies email addresses related to a domain by crawling the web.

Access Hunter IO

Use Cases

Understanding company email formats.
Gathering potential targets.
Enhancing OSINT reports with verified emails.

The Harvester

The Harvester is a built-in tool in Kali Linux for gathering emails, hostnames, subdomains, and more from public sources.

theHarvester -d example.com -b all -f emails

-d specifies the target domain
-b sets the source (ex: google, bing, linkedin, hunter, all)
-f Saves the results as emails.html and emails.xml in the working directory.

Use Cases

Quickly collecting all known public emails
Performing OSINT on target personnel
Identifying emails exposed via search engines or public APIs

Social Media Profiling

Gathering intelligence from social media platforms. This includes employee names, job roles, photos, events, and other contextual data that can be used for phishing or impersonation.

Public Job Postings

Job listings can reveal the technologies and software used within an organization. For example, a company hiring a “Splunk Engineer” likely uses Splunk internally.

Metadata Extraction

Publicly available files like PDFs or images may contain metadata such as usernames, software versions, timestamps, and internal paths. Tools: ExifTool, FOCA.

Google Dorking

Using advanced Google search operators to find sensitive information like exposed login pages, documents, backups, or misconfigured directories.

Search within a specific website: site:example.com
Finds pages related to "login" only on example.com: site:example.com login
Find subdomains of a site: site:*.example.com
Find specific types of files: filetype:pdf / ext:xls
Search keywords in the URL: inurl:admin
Search keywords in the body text: intext:"password"
Search entered keywords in the admin directory: intitle:"login"

Network Footprint Analysis

Building a profile of the organization's digital infrastructure from passive data: IP ranges, domains, hosting providers, and cloud services often using tools like Shodan or Censys.
